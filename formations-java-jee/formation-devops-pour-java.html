Formation DevOps pour Java animée par Cyrille Le Clerc

Programme

RÉSUMÉ

Comme le mouvement Agile a rapproché donneurs d’ordres et équipes de réalisation autour d’une vision commune orientée « produit », le mouvement DevOps rapproche aujourd’hui les équipes de développement (DEV) et d’exploitation (OPS) autour d’une vision commune orientée « service », afin de mieux concilier réactivité et qualité de service.

Cette formation présentera aux stagiaires des modèles DevOps et leur apprendra à les adapter à leur organisation.

SUJETS ABORDÉS

  • Historique du mouvement DevOps, les origines, les influences
  • Les processus et étapes éligibles à des fonctionnements DevOps : usine logicielle, livraison, déploiement, exploitation et troubleshooting
  • DevOps, comment placer le curseur entre Dev et Ops
  • Alignement des Dev aux réalités des Ops : rendre son application “prête pour la production”
  • Alignement des Ops aux enjeux des Dev : intégration de la plateforme de production à l’usine logicielle
  • Le rôle des outils dans les organisations DevOps, comment les choisir
  • Planète DevOps : les tendances, les mouvement émergents, Continuous Delivery, Infrastructure as Code et le Cloud

Bio

Cyrille Le Clerc

Après six années passées chez IBM Global Services, Cyrille a rejoint Xebia pour participer à l’essor des architectures Java légères et des méthodes Agiles.

Convaincu par le modèle “you build it, you run it ©” popularisé par Amazon, Cyrille intervient durant ses missions autant sur les phases de conception au contact des équipes fonctionnelles qu’auprès des équipes de développement et des équipes d’exploitation pour assurer le support niveau 3 ou réaliser les mises en production.

Cyrille est membre du projet Apache CXF, un participant actif du Paris Java User Group et auteur de nombreuses publications et notes techniques autour des problématiques de production Java/J2EE.
